Ingest Pipeline Flow (per source)

FETCH Call ATS API, get raw job list fetcher.fetch(slug)
MAP Normalize to unified JobCreate schema mapper.map(raw_job)
DEDUPE Remove duplicate external_job_ids within snapshot dict[external_job_id] = payload
EXISTING MAP Query Firestore: which jobs already exist for this source? db.collection("jobs").where("sourceId", "==", src)
STAGE Build insert/update dicts, sync blobs to GCS batch.set() or batch.update()
PERSIST Batch write jobs to Firestore (500 per batch) WriteBatch.commit()
LOCATION SYNC Parse location hints, resolve via GeoNames, denormalize into job docs batch.update(locations=...)
FINALIZE Close jobs not in snapshot: query open + lastSeenAt < syncStart, set closed query + batch.update(status="closed")
PUBLISH Send job IDs to Pub/Sub for async enrichment pubsub.publish("jobs-to-enrich", job_ids)
ENRICH (async) Cloud Run worker: LLM structured JD extraction + embedding generation llm.parse(desc) + embed(text) → Firestore update

Key Insights

  • Vertex AI Vector Search is 98% of Option A's cost — a single always-on node at ~$0.75/hr is $548/month. Overkill for 200k vectors.
  • Firestore native vector search is included — no additional cost. Supports 768-dim embeddings with cosine distance.
  • Pinecone Serverless free tier covers 2M read units + 5GB storage — likely sufficient for 200k vectors and 1k queries/day.
  • Cloud Run scales to zero — you only pay when requests are being served or jobs are running.
  • LLM enrichment cost (SiliconFlow Qwen) is ~$18 for a full corpus run — same regardless of architecture choice.
